jQuery is a JavaScript library.

What is a library?

What's a library?

A collection of reusable methods for a particular purpose.

If we had a math library, it might have functions like:

math.sum(array);
math.pow(num, num);
math.factorial(num);

Including a library

  • Include a script tag to the library on your page
  • Call functions from that library
<script src="http://imagine-it.org/math.js"></script>
<script>
  var answer = math.sum(2, 2);
  alert(answer);
</script>

We include the library before our JavaScript so that it loads before our code does.

Why use jQuery?

Why use libraries?

Why use libraries?

Write less code for common tasks.

Abstract on top of cross-browser differences.

Libraries can provide us with code that does all of this for us.

Why jQuery?

jQuery makes it easier for us to write code that is:

  • Shorter
  • Cross-browser compatible
  • Way less work


Don't believe me? We'll see...

jQuery vs. JavaScript

No library:

var elems = document.getElementsByTagName("img");
for (var i = 0; i< elems.length; i++) {
  elems[i].style.display = "none";
}

jQuery:

$('img').hide();

jQuery vs. JavaScript

No library:

var p = document.createElement('p');
p.appendChild(document.createTextNode('Welcome!'));
p.style.cssFloat = 'left';
p.style.backgroundColor = 'red';
p.className = 'special';
document.querySelector('div.header').appendChild(p);

jQuery:

var newP = $('<p>Welcome!</p>');
newP.css({'float': 'left', 'background-color': 'red')
newP.addClass('special');
$('div.header').append(newP);

Basic syntax

Breaking it down

Arguments and reading documentation

Basic syntax

We start with an HTML element:

<p>Welcome to jQuery!</p>

Use jQuery to add a class to it:

$('p').addClass('special');

And tada! Our HTML now has a class:

<p class="special">Welcome to jQuery!</p>

Breaking it down

We have one simple line of jQuery here, but there's a lot happening.

$('p').addClass('special');

$

$

$ is the global jQuery function.

You can also use jQuery if you want, but $ is the standard.

('p')

$('p')

('p') finds DOM element(s) according to what's in the quotes. In our example, that's all p elements.

This returns a "jQuery collection" of all p elements in the DOM.

addClass()

$('p').addClass('special');

.addClass() is a built-in jQuery method that adds the specified class to the collection.

It takes one argument: a string that is the class name (special in our example).

Read the docs here.

$(document).ready()

We don't want our code to run unless the DOM is ready for it. jQuery has a special function for this:

$(document).ready()

Using our existing example, we would write:

$(document).ready(function() {
  $('p').addClass('special');
});

This will add the class special to all of the p elements.

The semicolons are important because we want this code to run as soon as our browser reads it.

Getters & Setters

Find and change elements

With one line of code!

Getters

Getters return information about the HTML.

Sometimes they take arguments to narrow our focus:

<a href="http://www.google.com">Google</a>
$('a').attr('href');
// "Welcome!"

And sometimes they don't:

<p>Welcome!</p>
$('p').html();
// "Welcome!"

Setters

Setters change the HTML.

They almost always take an argument (sometimes more than one).

<p>Welcome!</p>
$('p').html("Hello");

Will change your HTML to:

<p>Hello</p>

Getters

Reading elements

HTML, attributes, and style properties

Reading elements

There are a few different ways to read elements.

Starting with this HTML:

<a href="http://www.yahoo.com" style="color:purple">Yahoo</a>

We can find out lots of fun things!

html()

Gets the HTML contents of the first element that matches your selector.

<a href="http://www.yahoo.com" style="color:purple">Yahoo</a>
console.log($('a').html());
// "Yahoo"

Read the docs.

attr()

Gets the value of a specific attribute for the first element that matches your selector.

<a href="http://www.yahoo.com" style="color:purple">Yahoo</a>
console.log($('a').attr('href'));
// "http://www.yahoo.com"

Read the docs.

css()

Gets the value of the style property for first element that matches your selector.

<a href="http://www.yahoo.com" style="color:purple">Yahoo</a>
console.log($('a').css('color'));
// "purple"

Read the docs.

Setters

Change elements

HTML, attributes, and style properties

Changing elements

We can change different parts of our HTML using jQuery.

Starting with this HTML:

<a href="http://www.google.com" style="color:purple">Google</a>

html('value')

Sets the value of the HTML.

<a href="http://www.google.com" style="color:purple">Google</a>
$('a').html('Yahoo!');

Changes the HTML to:

<a href="http://www.google.com" style="color:purple">Yahoo!</a>

Read the docs.

attr('attribute', 'value')

Sets the value of the property for the first element that matches your selector.

<a href="http://www.google.com" style="color:purple">Yahoo!</a>
$('a').attr('href', 'http://www.yahoo.com');

Changes the HTML to:

<a href="http://www.yahoo.com" style="color:purple">Yahoo!</a>

Read the docs.

css({'property': 'value'})

Sets the value of the style property for first element that matches your selector. Note the syntax difference here!

<a href="http://www.google.com" style="color:purple">Yahoo!</a>
$('a').css({'color': 'red'});

Changes the HTML to:

<a href="http://www.yahoo.com" style="color:red">Yahoo!</a>

Read the docs.

Loops

Many elements

Finding, changing, and modifying groups of elements

Loops: review

Loops allow us to call a method on every item in a group efficiently.

With jQuery, we have a method called each that we can use.

You can use each with an anonymous function, or with a named function.

Anonymous functions

each can take an anonymous function:

var links = $('a');

links.each(function(index, elem) {
  var link = $(elem);
  link.html(link.html() + '!');
});

Named functions

Or a named function:

var processElem = function(index, elem) {
  var link = $(elem);
  link.html(link.html() + '!');
};

var links = $('a');

links.each(processElem);

Events

Making changes after user action

Clicks, hovers, and more

Lots of events

Event documentation

There are tons of events.

Seriously, there are a ton. Don't try and memorize them.

A few of the popular events are...

Mouse events

  • Clicking: click(), actually clicking on something with your mouse
  • Double clicking: dblclick()
  • Hovering: hover(), mouse entering and then leaving an element
  • Entering and exiting: mouseenter() and mouseleave()


You can use all of these as their own functions, or with on(). Most people use them as their own functions, which is what we are going to practice.

click()

With an anonymous function:

$('a').click(function (event) {
  event.preventDefault();
  console.log('Not going there!');
});

Wait, wat.

What is event.preventDefault()?

preventDefault()

When you click something, a default behavior happens. If it's a link, that behavior is going to where the href points to.

event.preventDefault(); tells jQuery to not do the default behavior: the link doesn't take you anywhere, you stay exactly where you are on the page.

click()

click() can also be used with a named function:

function onButtonClick() {
  console.log('clicked!');
}

$('#button').click(onButtonClick);

Form events

Form specific:

  • Submitting a form: submit(), when you submit the form ("enter" key, or clicking a submit button)


Mostly used with forms, but not form specific:

  • Blurring: blur(), cursor leaves a form field or element
  • Focusing: focus(), cursor enters a form field or element; this is also used to help manage screenreaders and what they focus on after events and actions

submit()

Looks a lot like click(), doesn't it?

$('form').submit(function (event) {
  event.preventDefault();
  console.log('Not submitting, time to validate!');
});

Just like with click, we are preventing the default behavior again. The behavior we're preventing is form submission.

You can also use submit() with a named function.

Effects & Animation

Fun pretty things

Adding a little flair to our page

fadeIn()

You can make pretty things happen automatically:

$('#error').fadeIn();

Or after an event:

$('a').click(function (event) {
  event.preventDefault();
  $('#error').fadeIn();
});

Documentation

Effect options

Most effects and animations have multiple options that can be passed as arguments. You can control things like timing, delay, and speed.

It is best to consult the documentation for the method you want to use.

fadeIn() with options

These options control duration (default: 400 ms), easing (default: "swing"), and a function to do upon completion.

$("#menu").click(function() {
  $("#nav").fadeIn(200, "linear", function() {
    console.log("Faded in!");
  });
});
